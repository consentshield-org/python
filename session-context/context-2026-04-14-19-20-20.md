## Key Technical Decisions

### Stack
- **Next.js 16.2.3** over 14 (user corrected: always latest stable with security patches)
- **Bun.js** for dev tooling (per `~/.claude/CLAUDE.md` rule)
- **Turbopack** as default bundler (Next 16 default)
- **proxy.ts** replaces middleware.ts (Next 16); runs on Node.js runtime, not Edge
- **Tailwind CSS v4** with `@tailwindcss/postcss`, CSS-first config
- **Supabase** for Postgres + Auth; three scoped roles (`cs_worker`, `cs_delivery`, `cs_orchestrator`) exist at Postgres level but **REST API requires service_role key** — documented in cerebrum
- **Cloudflare Worker** with zero npm dependencies (policy), Web Crypto API for HMAC

### Architecture
- All 32 tables have RLS enabled. 11 verification queries pass on every deploy.
- Buffer tables are append-only for authenticated users; writes only via scoped service roles.
- Immediate deletion of buffer rows on confirmed delivery (5-min sweep as safety net, 1-hour stuck detection, 24-hour P0 escalation).
- Signed callback URLs for deletion receipts: `HMAC-SHA256(receipt_id, DELETION_CALLBACK_SECRET)`, timing-safe verify.
- Per-org encryption key: `HMAC-SHA256(MASTER_ENCRYPTION_KEY, org_id + encryption_salt)` — used for integration_connectors.config (bytea via pgcrypto RPC).
- HMAC-SHA256 on consent events: `HMAC(org_id + property_id + timestamp, event_signing_secret)` with ±5 min timestamp window and previous-secret fallback during 1-hour grace period after publish.

### Workflow
- ADR-driven: no code without an ADR, no sprint without tests, no merge without a changelog.
- Per-area changelogs (schema, worker, dashboard, api, edge-functions, infra, docs).
- Documented reviews (`docs/reviews/`) required before promoting design docs to source-of-truth.
- `MASTER.md` index for the memory system and `.wolf/` for OpenWolf context management are both active.

## Bugs Encountered and Fixed

1. **pgcrypto's `gen_random_bytes()` needs `extensions.` prefix on hosted Supabase** — added to cerebrum.
2. **Supabase REST API doesn't support custom Postgres roles** (only anon and service_role JWTs) — Worker uses service_role key with app-level query restriction.
3. **React 19 purity rule blocks `Date.now()` and `new Date()` inline in server components** — extracted to `lib/compliance/score.ts` helpers (`isoSinceHours`, `nowIso`, `daysBetween`).
4. **Next.js 16 removed `next lint`** — script changed to `eslint src/` directly.
5. **Next.js 16 requires Suspense around `useSearchParams`** — wrapped LoginForm.
6. **Supabase join type mismatch** for dashboard org query — separated into two queries.
7. **tsconfig picked up worker/ and supabase/functions/** — both excluded to stop Next.js type-checking Deno imports.
8. **Banner-test.html origin mismatch** (`file://` vs allowed_origins) — served via `python3 -m http.server 8080` after adding `http://localhost:8080` to property's allowed_origins.
9. **DNS propagation lag** — `.in` registry had Cloudflare NS in WHOIS, but public resolvers cached GoDaddy's NS. Waited ~30 minutes for propagation.
