Kill switches
All normal
banner_delivery
When engaged, Cloudflare Worker returns a no-op banner.js for ALL customers. Use only when banner code is dangerous (security incident, malicious change). Customer sites continue to function.
● Normal
depa_processing
When engaged, the process-consent-event Edge Function short-circuits and returns 200 without creating consent_artefacts. Triggers + safety-net cron continue but do nothing. Use only if DEPA logic is producing bad artefacts at scale.
● Normal
deletion_dispatch
When engaged, orchestrate-deletion does NOT dispatch new deletion_requests to connectors. In-flight requests continue. New revocations queue but don't fan out. Use during connector-side outages or when deletion logic itself is broken.
● Normal
rights_request_intake
When engaged, /api/public/rights-request returns 503 Service Unavailable. Use during major abuse incidents (rate-limit alone insufficient) or DPB regulatory takedown order.
● Normal
Engaging a kill switch requires reason ≥ 10 chars (audit-logged). Confirmation prompt asks the operator to type the switch_key to confirm. Worker + Edge Functions read flag state from Cloudflare KV; sync runs every 2 minutes via the admin-sync-config-to-kv cron.
