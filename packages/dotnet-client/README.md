# ConsentShield .NET SDK

Official .NET 8 client + ASP.NET Core integration for the [ConsentShield Compliance API](https://api.consentshield.in/v1).

- **NuGet**: `ConsentShield.Client.AspNetCore` 1.0.0 (ASP.NET Core integration)
- **NuGet**: `ConsentShield.Client` 1.0.0 (raw client; non-ASP.NET callers)
- **Source**: <https://github.com/SAnegondhi/consentshield-dotnet>

## Install (ASP.NET Core)

```bash
dotnet add package ConsentShield.Client.AspNetCore --version 1.0.0
```

```jsonc
// appsettings.json
{
  "ConsentShield": {
    "ApiKey": "cs_live_...",
    "FailOpen": false
  }
}
```

```csharp
var builder = WebApplication.CreateBuilder(args);
builder.Services.AddConsentShield(builder.Configuration);

var app = builder.Build();

app.MapPost("/api/marketing/send", async (UtilityApi api) =>
{
    // verify, gate, return
    return Results.Accepted();
});

app.Run();
```

## Compliance contract

| Outcome | Default behaviour |
|---|---|
| 4xx response | `ConsentShieldApiException` — ALWAYS surfaces, never retried |
| Per-attempt timeout | `ConsentShieldTimeoutException` — NEVER retried |
| 5xx + transport failure (fail-CLOSED, default) | `ConsentVerifyException` with `VerifyFailureCause` |
| 5xx + transport failure (fail-OPEN) | open outcome with cause set |

Retry budget: 3 retries on 5xx + transport with 100 / 400 / 1600 ms exponential backoff. Per-attempt timeout 2 s. Configurable via `ConsentShield:Timeout` and `ConsentShield:MaxRetries`.

## Source layout

- `generated/` — generated by `openapi-generator-cli@v7.10.0` from `app/public/openapi.yaml` in the upstream monorepo. Do NOT hand-edit.
- `wrapper/ConsentShield.Client.AspNetCore/` — DI extension + `RetryHandler` `DelegatingHandler` + exception hierarchy.
- `wrapper/ConsentShield.Client.AspNetCore.Tests/` — xUnit test suite.
